# CLAUDE.md - Net Worth Tracker (Lean)

## Project Overview
Net Worth Tracker is a Next.js app for Italian investors to track net worth, assets, cashflow, dividends, performance metrics, and long-term planning with Firebase.

## Current Status
- Stack: Next.js 16, React 19, TypeScript 5, Tailwind v4, Firebase, Vitest, Framer Motion, Recharts, Yahoo Finance, Borsa Italiana scraping, Anthropic
- Latest (2026-06-05): **Budget redesign (issue #148).** Opt-in budgets (only created budgets shown — `reconcileBudgetItems`, no per-category auto-fill) with create/edit/delete via `ResponsiveModal` + debounced **auto-save** (`useBudgetConfig`, 800ms, **paused while the allocation is invalid**). **Overall budget** = ceiling on ALL month spending (`getMonthlyTotalExpenses`, not just budgeted) with `validateBudgetAllocation` (Σ monthly category ≤ overall). **Income budgets** as separate targets (inverted progress). Per-item **period** `monthly`|`annual` — annual tracked YTD (spiky categories like vacations), list split Mensili/Annuali. **Spending Forecast** + **Insights** + in-app threshold **Alerts** (50/75/90/100% + forecast overrun). Forecast/insights **dampened early-month** (`buildSpendingForecast(..., referenceMonthlyAverage)` blends actual vs previous-year pace; `MIN_FORECAST_DAYS` gate; insights compare vs `expectedSpendToDate`). Pure layer `lib/utils/budgetUtils.ts` (validate/forecast/insights/alerts, tested); monolith split into `components/cashflow/budget/*`. `monthlyAmount`→`amount` rename with read-migration. Alerts also surface in the **monthly summary email** + new **weekly budget email** (Sundays, +1 AI sentence, `lib/server/weeklyBudgetEmailService.ts`, Cron Phase 6, `weeklyBudgetEmailEnabled` toggle + test-send). Persistence gotcha fixed (never reconcile vs empty categories). Also refreshed impeccable prompt docs (`docs/{critique,audit,shape,polish}-prompts.md`) for Assistant + Allocazione.

## Known Issues (Active)
- FX conversion depends on Frankfurter API availability; cache fallback (24h TTL) is used on failure — pre-migration non-EUR assets (USD, CHF, etc.) without `currentPriceEur` will show native price as EUR until first price update. GBp assets are safe: the pence→GBP fallback (`currentPrice / 100`) prevents the 100× inflation even without `currentPriceEur`.
- Demo account requires manual Firebase setup (create user, populate Firestore with realistic fake data, set three env vars)

## Design Context

### Users
Italian self-directed investors who want to understand their financial position quickly and confidently.

### Brand Personality
Elegant, sophisticated, personal.

### Aesthetic Direction
Linear / Vercel inspired clarity with polished motion, dense but readable data presentation, and strong dark mode quality.

### Design Principles
1. Data first, decoration second
2. Motion with purpose
3. Density is a feature
4. Precision builds trust
5. Personality lives in the details
